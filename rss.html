2/8-9の日程で、群馬県は 上野村 の天体観測イベントに参加してきました。

参加の経緯

私が FaceBook で “天体観測したい” とつぶやいたのがきっかけで、 NatureService というNPO法人が動いてくれて、 気がついたら村公認のイベントになっていました。

本当は知人数名でこっそりやるつもりだったのでちょっと動揺しましたが、 まぁ、そういうのもありかな、と思って参加しました。

内容

群馬県上野村の ベストスポット で、とにかく快適に天体観測をしよう、という趣旨のイベントです。

真冬の天体観測を経験したことのある人はわかると思いますが、外は滅茶苦茶寒いです。 寒いので、野外に炬燵を設置して、ヌクヌクしながら星を見ようという画期的なイベントです。 あと、寝たいときに寝られるように、アメリカ式の本格的なキャンピングカーも準備してあります。

それから、近くに 不二洞 という関東最大の鍾乳洞があったので、直前に鍾乳洞ツアーも付け加えました。

参加メンバー

半分位は以前のNatureServiceのイベントで面識のある人で、 残りの半分位は私の知らない人。 20台前半から40台くらいまでの男女12名が集まりました。 ほとんど全員が一眼を持っていて、フルサイズを持っている割合もかなり多かったです。

せっかくなので良い写真取りたいですもんね。

成功談

  • 鍾乳洞、期待以上でした。さすがは関東最大。見にいきたいと主張して良かった～。
  • 星、滅茶苦茶綺麗でした。綺麗な星空を見ると、なにか圧倒されたような気分になります。 東京から結構近い場所なのに、すごいです。あー、もう一回行きたい～。
  • 木星見ました。衛星4つと縞模様もはっきり見えました。 小学校の頃、望遠鏡を使って、自宅のベランダから月や土星、木星などを観測していたので、とても懐かしい気分になりました。
  • メンバーが個性的で面白かったです。今回のイベントが成功したの最大の理由はこれでしょう。

失敗談

その1

天体望遠鏡 を自費で借りて担いで持っていったのですが、 想像以上の大きさだったので、集合場所にたどり着いた時点でヘトヘトでした。(しかも遅刻)

望遠鏡を担いで駅の階段から降りてくる絵は確実に面白かったと思うのですが、 余裕がなかったので写真もビデオもありません。

その2

撮った星野写真はこの一枚...

まともにカメラの設定もしてないです...。酔っていたのでしょうがないのです。 寒い中炬燵が出てて、料理と酒があったら、そりゃ飲みますよ。。 飲んだら寝ちゃいますよ。。

鍾乳洞はとりました。

その他・雑感

私はm3/4のミラーレスを持っていったのですが、やはり高感度撮影ではフルサイズやAPS-Cのカメラの足元にも及びませんね...。 これはオーロラを見に行ったときにも痛感したことではありますが。。

ただ、レンズも含めて小型の方が私は好きなので、3/4のセンサーの性能が十分に向上するのを待つか、サイズを諦めてフルサイズ機を買ってしまうか、悩んでいます。

まとめ

  • 天体観測を本気でしたいのであれば、飲むべきではない。
  • 鍾乳洞がとてもよかった
  • こういうイベントは、企画内容もそうですが、メンバーが重要ですね。今回は大当たりでした!

]]> Sat, 02 Feb 2013 00:00:00 +0900 http://blog.0-8.in/2013/01/12/multi.html http://blog.0-8.in/2013/01/12/multi.html <![CDATA[UNIXコマンドを並列化する]]> UNIXコマンドを並列化する

もう一年近く前、会社の方が “UNIXのテキスト処理関連のツールをマルチスレッド対応してほしい” とつぶやいていました。

「マルチスレッドは面倒くさいけど、マルチプロセスなら簡単だよ!」ってことで、プチフレームワークを書いたのが去年の5月。

それを最近になって思い出して、 github にあげたので紹介します。

何をするもの?

テキスト処理をしていると、簡単なクレンジング処理などで 行単位で並列化出来るような処理がたくさんあります。 multi は標準入力を1行毎に区切って複数のプロセスに渡すことでテキスト処理の高速化を図るツールです。 テキストを分割したりプロセス管理したりするので、重い処理をしない場合は遅くなることがあります。

サンプル

以下のコードでは行毎に hoge.txt を10個の mecab にかけています。

cat hoge.txt | multi -m "mecab" -n 10

オプション

指定できるオプションは以下のとおり。

usage: multi [OPTIONS]
-c   combiner command
-h   show this message
-m   mapper command (required)
-n   number of mappers (required)
-s   splitter command
-S   use internal splitter/combiner which preserve the order of the input sequence

まず、言葉の定義ですが、標準入力を1行毎に区切る部分を splitter、 それを受け取って具体的な処理をする部分を mapper、 処理結果を一つにまとめて出力する部分を combiner と呼ぶことにしました。 それぞれ、 -s、 -m、 -c でコマンドを指定することができます。

splitter と combiner は必須オプションではなく、 省略すると行ごとに区切って、空いている mapper にわたし、 mapper の出力を早いもの順で結合して出力します。 combiner にコマンドを指定する場合は、 入力ファイル名に対応する部分に {} を書きます。 例えば

cat hoge.txt | multi -m "sort" -c "sort -m {}"

とすると {} の部分が mapper(-m “sort” の部分) の標準出力のリストに置換されます。

また、行の順序を保ちたい場合には -S オプションを利用します。この場合、 splitter や combiner は指定できません。

インストール

インストール手順は以下のとおりです。

  1. github よりダウンロード
git clone https://github.com/ohtaman/multi.git multi
  1. configure & make
cd multi
./configure
make
sudo make install
